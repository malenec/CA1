  • Who was active during this sprint?
    • All 4 members of the group have been 100% active during this sprint.
  • Who did what, during this sprint?
    • We have all 4 been involved in every single step together during this sprint. We have taken turns at being the one who does the coding and at being copilots. This has worked perfectly during the first iteration as this sprint solely entailed set up of the overall system and therefore was not obvious to divide between us. We have also been able to solve issues quickly with the shared amount of knowledge between all of us - there was always at least 1 person who remembered an important detail from class or previous projects, that was the key to the solution. This has made the sprint run smooth and fast from beginning till end.
  • What did your team accomplish during this sprint?
    • Domain model and class diagram has been designed
    • KanBan using Trello has been created, discussed, structured and color-coded as well as updated continuously and last but not least screenshot at the beginning and end of the sprint
    • User stories have been written
    • Estimation game has been played and all user stories have been given a t-shirt size estimate as a result of this game
    • Startcode has been set up
    • Database including all data from cityinfo and hobby scripts has been set up and implemented
    • CI pipeline has been set up and is running smoothly
    • Entities have been generated from the database using JPA and associations as well as add methods have been properly created and filled out

    • Who was active during this sprint?
      • Again all 4 members of the group have been 100% active during this sprint.
    • Who did what, during this sprint?
      • We have used the same strategy as for the previous sprint. We have again all 4 been involved in every single step together during this sprint, and we have again taken turns at being the one who does the coding and at being copilots. During this sprint we have made this choice, because the code has caused us so many headaches - all four of us - that we have had no other option than to have all men on deck to solve the issues at hand. We have been surprised at how many problems we’ve run into, and every single time without exception the solution has been a minor detail that needed to be changed in order for everything to work as expected. But this has not been due to an overlook over something silly like a semicolon etc., but very fundamental things caused by holes in our understanding of these new areas and tools we’ve been introduced to this semester.
    • What did your team accomplish during this sprint?
      • We have NOT completed anywhere near all tasks suggested for the second iteration, if the expectation was that the backend was pretty much done this evening - but we have made managed to make GET, POST and PUT endpoints, which was perhaps the most important goal to accomplish during this sprint:
      • Database has been redesigned
      • Entities have been regenerated (A LOT of times)
      • DTOs have been created for all entities
      • Tests have been generated before methods (TDD - which we are proud of during all the other challenges we’ve fought through)
      • Populator has been created
        • getPersonbyId
        • addHobbyToPerson
        • addPerson
        • updatePerson
        • getAllPersons
        • getAllHobbies/li>
        • getHobbyByID
      • Database has been filled up with sample data for persons including all possible information a person can have like address, phones and hobbies
      • The following methods have been created
      • Resources for each method have been created
      • Endpoints for each method have been created
      • KanBan using Trello has been further color-coded as well as updated continuously and again last but not least screenshot at the beginning and end of the sprint
      • Updated diagram has been generated for each package at the end of the sprint
      • All relevant documentation has been deployed including this diary as well as an updated API documentation and all links that are asked of us to provide
